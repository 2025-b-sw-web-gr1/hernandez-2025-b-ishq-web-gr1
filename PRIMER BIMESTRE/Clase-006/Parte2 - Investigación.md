## 1. HTML5 (HyperText Markup Language)

### Descripción
HTML5 es la quinta y más avanzada versión del lenguaje de marcado que define la **estructura semántica de los documentos web**. Introduce etiquetas como `<header>`, `<article>`, `<section>`, `<nav>` y `<footer>`, que permiten describir el contenido de forma más significativa tanto para usuarios como para motores de búsqueda y tecnologías de asistencia. Viene incluido el soporte nativo para **audio, video y gráficos**, eliminando la dependencia de plugins externos como Flash. También habilita el uso de diversas APIs que permiten desarrollar aplicaciones web más dinámicas e interactivas.

### Justificación de la elección
HTML5 constituye la base de toda la Web moderna y es sumamente importante conocerla debido a que, es como el componente en donde se derivan más estilos. Es bastante fácil de aprender en cuanto a su implementación asegurando una mejora en la práctica en cuanto la accesibilidad, el posicionamiento en buscadores, la compatibilidad entre navegadores y la sostenibilidad del desarrollo web a largo plazo.

## 2. CSS (Cascading Style Sheets)

### Descripción

> El portal web w3School define CSS de la siguente forma: 

**CSS es el lenguaje que se utiliza para dar estilo a una página web.**

- **CSS** significa **Hojas de Estilo en Cascada**.
- **CSS** describe cómo deben mostrarse los elementos HTML en pantalla, en papel u otros medios.
- **CSS** ahorra mucho trabajo, ya que permite controlar el diseño de varias páginas web al mismo tiempo.
- Las **hojas de estilo externas** se almacenan en archivos **CSS**.

Un estandar encargado de definir la **presentación visual y el diseño** de las páginas web. Permite controlar colores, tipografías, márgenes, animaciones y la distribución de los elementos en la pantalla. Las versiones más recientes incorporan herramientas como **Flexbox y Grid**, que facilitan la creación de interfaces complejas y responsivas, adaptándose a distintos tamaños de pantalla y dispositivos.

### Justificación de la elección
Es una de las opciones, en lo personal, más utilizadas al momento de hacer un proyecto académico. CSS ha evolucionado al nivel de ser fundamental para ofrecer una **experiencia de usuario agradable y funcional**. Ya que te permite separar el contenido de la presentación,  además, tiene mucho contenido e información debido a su popularidad.

## 3. Web Components 

### Descripción
Web Components es un conjunto de estándares que permite crear **componentes web personalizados, reutilizables y encapsulados**, funcionando de forma nativa en el navegador. Está compuesto por:

#### Custom Elements

    Permiten crear nuevas etiquetas HTML personalizadas, con nombre, comportamiento y funcionalidad propios. Facilitan la construcción de componentes reutilizables que se integran de forma nativa en el navegador.

#### Shadow DOM

    Proporciona un árbol DOM encapsulado, aislando la estructura y los estilos de un componente del resto del documento. Esto evita conflictos de estilos y asegura que el componente funcione de manera consistente.

#### HTML Templates

    Permiten definir fragmentos de HTML reutilizables que no se renderizan inmediatamente. Estos fragmentos pueden clonarse e insertarse dinámicamente en el DOM mediante JavaScript, optimizando la reutilización y organización del código.


Estos componentes pueden integrarse fácilmente en cualquier proyecto web, independientemente del framework utilizado.

### Justificación de la elección
Aqui entra el concepto de **arquitectura web modular** y lo que se puede utilizar si quieres optar por esta alternativa, este ayuda a la reducción de la dependencia de librerías externas, lo que radica en interoperabilidad y reutilización del código en distintos proyectos.

## 4. RDF (Resource Description Framework) 

### Descripción
Este va más enfocado en la representación de información estructurada y relaciones entre datos en la Web. Utiliza un modelo basado en triples (sujeto, predicado y objeto), permitiendo que la información sea procesada e interpretada por máquinas. Definido en algunos sitios como la base de la Web Semántica y se utiliza en proyectos de datos abiertos, bibliotecas digitales, sistemas académicos y motores de búsqueda avanzados.

### Justificación de la elección
No es en lo absoluto planeado para un desarrollo tradicional, ya que está enfocado en desarrollar una Web más inteligente e interconectada, permitiendo reutilizar y vincular información a gran escala. Sigue la tendencia SOLID si se puede ver desde esa perspectiva, pienso que es importante ver los extremos de cómo puedes enfocar los proyectos y estos estandares te ayudan a ampliar la perspectiva.

## 5. WebAssembly (Wasm) 

### Descripción
WebAssembly es un estándar que permite ejecutar **código de alto rendimiento** dentro del navegador. Funciona como complemento de JavaScript y admite lenguajes como C, C++ y Rust. Wasm se ejecuta en un entorno seguro y controlado, ofreciendo un rendimiento cercano al de las aplicaciones nativas.

### Justificación de la elección
Amplía significativamente las capacidades de la Web, este permite desarrollar aplicaciones complejas como editores gráficos, videojuegos, simuladores y herramientas científicas directamente en el navegador.
